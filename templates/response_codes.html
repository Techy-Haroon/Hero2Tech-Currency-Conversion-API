Response Codes

Below is a list of common HTTP response codes returned by the API:

  • 200 OK: The request was successful, and the server responded with the requested data.
  • 400 Bad Request: The server could not understand the request due to invalid syntax or missing parameters.
  • 401 Unauthorized: Authentication failed. The API key provided is missing or invalid.
  • 403 Forbidden: The user has reached their request limit.
  • 500 Internal Server Error: An unexpected error occurred on the server. Please try again later.

Make sure to always include your API key in your requests, either in the Authorization header or as a URL parameter.

*If You Include API key in Authorization header, then no matter if you gave it as URL parameter, it is always ignored. {% include 'docs_go.html' %}
